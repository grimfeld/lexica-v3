# App-generated pronunciation (the one content exception)

Pronunciation (IPA) is the single place the app originates content rather than
the user. It is generated by a pipeline: dictionary lookup first, LLM fallback
when the word is missing. There is no user review step, because a user who does
not read IPA cannot validate it.

This is a deliberate exception to the project's north star (the app never
originates learning material). It is justified because pronunciation is
display-only and never tested, and because the user is typically unqualified to
author it. The risk — silently incorrect LLM-generated IPA the user cannot
catch — is accepted for MVP; a "report bad pronunciation" affordance may be
added later.

IPA is nullable. Note creation never blocks on pronunciation; the field is
backfilled asynchronously by a background job, so authoring stays decoupled
from network and LLM latency.
